Finetuning Plan

Failure Taxonomy

The evaluation reveals three mechanistically distinct failure regimes, each requiring a different training lever:

  • OCR-as-crutch. The model reads text labels instead of perceiving visual content. Value-label conflict: 0% accuracy, 100% text-reliant. Grid counting: 49% without text → 100% with text. This silently affects every chart/table/document task — when labels are correct, OCR-reliance masks the underlying blindness.
  • Memorized priors override vision. Board games: 100% bias alignment (always outputs 8×8 for chess, 19×19 for Go regardless of actual size). Patterned grid add-anomalies: 0%. Extended thinking (budget=2048) produces no improvement — confirming these are perceptual failures, not reasoning failures.
  • Perceptual resolution limits. Smooth degradation along psychometric curves: proximity threshold at ~12–15px, color discrimination degrading with ΔL, pie chart angular estimation at 53%, path tracing scaling with distractor count.

Three-Stage Training Pipeline

Stage 1: SFT Visual Grounding (~300K examples). Break the OCR dependency. Train on charts without value labels, grids without coordinate text, pie charts without percentage annotations. Every SFT target includes chain-of-thought referencing visual landmarks. Curriculum: weeks 1–2 no-label only → weeks 3–4 introduce wrong-label conflict detection → weeks 5–6 reintroduce correct labels. The reintroduction is critical: we want a model that verifies text against visual evidence, not one that ignores all text. Include 70K solved-task regularization examples.

Stage 2: DPO Bias Override (~75K pairs). Board games are the single best DPO source: 100% bias alignment with zero variance means every non-canonical instance produces a perfectly clean preference pair. Chosen: explicit counting ("1, 2, 3, 4, 5, 6, 7 — that's 7 rows → {7}"). Rejected: pattern-matching ("Chess board → {8}"). Safeguard: include canonical boards where both responses demonstrate counting. Additional sources: value-label conflicts (0%), add-anomalies (0%), touching circles at threshold (0%), color at low ΔL.

Stage 3: RL Perceptual Sharpening (~175K episodes). All tasks have exactly computable ground truth. Curricula start where the model succeeds (~90%+) and push into failure regimes: overlapping shapes (overlap 0.1→0.5), path following (2→6 connections), pie estimation (3→8 slices), dense chart reading (3→10 series).

SFT target (left image): "What is the value for Apr?"

"The gridlines are at 0, 20, 40, 60, 80, 100. The Apr bar extends just above the 80 gridline. My estimate: {82}"

DPO pair (right image): "What is the value for Apr?" (bar height ~82, label says "55")

Chosen: "I see a label '55' above the Apr bar, but comparing bar height to the y-axis, it clearly extends just above 80. The label appears incorrect. Based on bar height: {82}"
Rejected: "{55}" — the model's actual output, reading the label directly.

RL reward function (for no-label charts with ≥3 series):

R = 1.0 if within 5% of GT or ±2 (whichever larger)
R = 0.5 if within 2× tolerance
R += 0.1 bonus if response references visual landmarks ("gridline", "y-axis", "between")
Tolerance-based because visual interpolation between gridlines has irreducible noise — even humans estimate.

Why this data. No-label SFT forces the model to develop visual estimation strategies. Wrong-label DPO teaches conflict detection and override — the real-world failure mode (stale labels in recycled slide decks). Correct labels reintroduced in weeks 5–6 restore normal text+visual integration.
